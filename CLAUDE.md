pnpm run build        # Production build (outputs to /dist)

There is no lint or test script configured.

## Architecture

This is a single-page React 18 + Vite + TypeScript portfolio site. All navigation is **overlay-based** — there is no router. The entire app renders as one full-screen `Hero` component that manages four `useState` booleans (`showWork`, `showAbout`, `showContact`, `showPuzzle`). Each section (Work, About, Contact, UXPuzzleGame) slides in as a full-screen `motion.div` overlay animated by Framer Motion (`motion/react`). ESC and scroll-down trigger open/close.

### Key architectural points

- **`src/app/App.tsx`** is the root — it wraps `Hero` in `ThemeProvider` and renders `CustomCursor`.
- **`ThemeContext`** (`src/app/contexts/ThemeContext.tsx`) manages dark/light theme by toggling the `.dark` class on `<html>` and persisting to `localStorage`. Components read it via `useTheme()`.
- **Canvas backgrounds** (`StarfieldBackground.tsx` — 800-star HTML5 Canvas, `ThreeBackground.tsx` — wireframe sphere via raw canvas) are rendered only when no overlay is open. They use `requestAnimationFrame` loops that must be cancelled on unmount.
- **`UXPuzzleGame.tsx`** is a memory card game whose leaderboard is stored in `localStorage`.
- **`src/app/components/ui/`** is a full shadcn/ui component library — these are pre-built Radix UI wrappers and are largely unused by the portfolio itself; they exist because this project was generated by Figma Make.

### Path alias

`@` resolves to `./src/app` (configured in `vite.config.ts`). Use `@/components/...`, `@/contexts/...` etc.

### Figma Make integration

`vite.config.ts` includes a `figmaAssetResolver` plugin that maps `figma:asset/<filename>` imports to files in `src/assets/`. The React and Tailwind plugins are both required by the Make runtime — do not remove them.

### Styling

- **Tailwind CSS 4** via the `@tailwindcss/vite` plugin (not PostCSS). Config is inline in CSS, not `tailwind.config.js`.
- CSS entry point: `src/styles/index.css` → imports tailwind, `tw-animate-css`, `default_theme.css` (shadcn design tokens), and `globals.css`.
- Dark mode uses Tailwind's `dark:` variant triggered by the `.dark` class on `<html>` (defined as `@custom-variant dark (&:is(.dark *))`).
- **Brand palette** (use these, not the shadcn token names, for portfolio UI):
  - Light background: `#d4d0c8` / Dark background: `#1a1816`
  - Text primary: `#2a2520` / Secondary: `#4d4339`
  - Accent gold light: `#87682A` / Accent gold dark: `#c9a961`

### Customizing content

| What | File |
|------|------|
| Personal info, nav | `src/app/components/Hero.tsx` |
| Projects/portfolio | `src/app/components/Work.tsx` |
| Bio, experience, skills | `src/app/components/About.tsx` |
| Email, social links | `src/app/components/Contact.tsx` |
| Page title, meta tags | `src/app/index.html` |

### Notes

- The IP address displayed on the hero is fetched live from `https://api.ipify.org` — this is the one external network call the app makes.
- The app uses `pnpm` exclusively; do not use `npm` or `yarn`.
